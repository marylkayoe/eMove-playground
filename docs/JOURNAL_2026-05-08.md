# Journal 2026-05-08

## Acceleration Primitive Events During LAR

The primitive-event work was consolidated into a valley-delimited compound-bout
pipeline and accompanying handoff documentation.

The main conceptual update is that a compound movement is no longer treated as
any group of peaks close in time. Visual inspection showed that close peaks
separated by a deep recovery valley should usually be treated as separate
movement episodes. The current working definition is therefore:

```text
compound bout = valley-delimited movement episode with two or more subpeaks
```

Adjacent subpeaks are split if the intervening valley drops below:

```text
0.50 * min(leftPeak, rightPeak)
```

The current decomposition uses:

- primary event threshold: `4 * median(noiseSigma)`,
- lower compound-subpeak threshold: `2 * median(noiseSigma)`,
- compound subpeak minimum distance: `0.35 s`,
- compound search window around anchor: `[-1.5, +4.5] s`.

The main detector works on:

```text
eventSignal = max(motionEnvelope - localBaseline, 0)
```

For original-envelope trace figures, the detector-equivalent threshold is:

```text
localBaseline + 4 * median(noiseSigma)
```

The helper `plotEnvelopeEventsWithNoiseBand.m` was added for reusable example
trace figures with:

- original motion envelope,
- detector-equivalent shaded background/noise region,
- unitary primary peaks,
- compound-bout subpeaks,
- portable `.fig` saving behavior.

Current grant-oriented Waseda summary:

- usable bouts: `735`,
- unitary bouts: `203`,
- compound bouts: `532`,
- fraction compound: `0.724`,
- median compound subpeaks: `3`,
- median compound active span: `1.184 s`,
- median within-compound interval: `0.608 s`.

Current physical-support summary:

- median gravity-corrected event/random RMS ratio: `1.326`,
- median filtered-magnitude event/random RMS ratio: `1.423`,
- median eventSignal event/random RMS ratio: `2.340`,
- fraction with gravity-corrected event/random RMS ratio above 1: `0.770`.

The best current interpretation is:

> LAR micromotion includes repeatable acceleration-supported primitive-like
> events, and many larger valley-delimited movement bouts contain multiple
> such elements arranged with non-random subsecond timing.

This should not yet be promoted to:

> LAR consists of confirmed physiological motor primitives.

The next dataset-level questions for Simo's controlled experiments are:

- whether event rate differs by subject, context, or interoceptive state,
- whether compound-bout fraction differs by context/state,
- whether compound-bout duration differs by subject or state,
- whether secondary subpeak timing shifts by context/state,
- whether accelerometer-defined events can also be identified in
  motion-capture marker trajectories.

## Event-Class Pipeline Update

The folder-level Waseda primitive-event pipeline now carries the same
valley-delimited unitary/compound bout definition into condition and subject
comparisons. The reproducible entry point is:

```zsh
/Applications/MATLAB_R2024b.app/bin/matlab -batch "run('scripts/run_waseda_event_class_pipeline.m')"
```

The script resolves the magnitude files from common local/Dropbox locations,
or from `WASEDA_ACC_MAGNITUDES` if set. It writes full generated outputs under:

```text
scratch/waseda_event_class_pipeline_20260508_onset_only/
```

and promotes the current curated figure pairs to:

```text
figures/WASEDA_ACC_EVENT_CLASSES_20260508/
```

Promoted figures:

- `primitive_event_class_pipeline_event_class_grouped_mean_waveforms.png/.fig`
- `primitive_event_class_pipeline_cdfs_by_condition_and_event_class.png/.fig`
- `primitive_event_class_pipeline_cdfs_by_subject_and_event_class.png/.fig`

All average waveform panels in this promoted set are onset-aligned, not
peak-aligned. Unitary bouts align to the estimated onset before their only
subpeak. Compound bouts align to the estimated onset before the first same-bout
subpeak. The waveform figure includes both raw-amplitude means and per-event
peak-amplitude-normalized means. Peak-aligned mean-waveform figures are no
longer generated by `analyzePrimitiveEvents`.

Validation on the current Waseda magnitude files:

- unitary anchors: `205`,
- compound anchors: `534`,
- onset-aligned waveform snippets after boundary-window exclusion:
  `204` unitary and `534` compound.
