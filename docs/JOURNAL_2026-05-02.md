## Baseline branch: what finally felt conceptually right

The most important correction today was not numerical. It was conceptual.

I kept trying to ask the baseline traces for a sparse-event story:

- candidate disruptions
- rare exits
- timing buildup

That was not completely wrong, but it kept producing weak versions of the
claim I wanted.

The traces themselves were pointing somewhere else.

What they seemed to show was not necessarily:

- "more events"

but more like:

- less local steadiness
- more wobble
- more drift
- more switching between calmer and less calm subsections

Once framed that way, the analysis became more coherent.

### What changed

1. structural event filtering was still worth doing
   - mostly as cleanup
   - mostly to learn which structural criteria were too harsh

2. brief exits from the low-animation regime were a better object than raw
   phasic peaks
   - but still only weakly late-skewed

3. the first result that actually matched the visual gut was rolling
   instability
   - especially position drift
   - and more modestly speed roughness

### What I should remember

- speed-only event detection can miss the phenomenon if the real effect is
  nonstationarity rather than large movement
- position is not redundant with speed in practice because speed compresses
  structure that can matter for local stability
- "mild late-trial instability" is a much more defensible story than
  "strong buildup of phasic baseline disruptions"

### What not to forget

The effect is still modest.

This is not the point where I should start writing inflated prose about
baseline destabilization as if it were already proven.

But it *is* the first version of the baseline question that feels like it is
tracking the same thing the traces seem to be showing.
