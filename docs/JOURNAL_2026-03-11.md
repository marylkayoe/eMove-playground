# Journal Notes (2026-03-11)

## 1) What Settled Today

- Cross-repo reproducibility is now clear:
  - external legacy CODE and playground CODE produce identical KS outputs when subject sets are matched.
  - apparent drift came from inclusion policy mismatch, not from algorithmic mismatch.
- Manifest run path is stable enough for full-cycle execution (batch metrics -> CDF -> KS heatmap -> stick figure).
- Stick-figure plot is now usable again with current marker-group schema.

## 2) Practical Lessons

1. Every figure/report should state:
   - subject inclusion mode (`applySubjectExclusions` true/false),
   - `minSamplesPerCond`,
   - excluded emotion groups (if any).
2. Visual confusion happens quickly when defaults are implicit.
3. The dataset is sensitive to sample gating: wrist groups disappear at high `minSamplesPerCond`, then come back at 200.

## 3) Visualization Ideas For Next Iteration

### A) Session-Level QC Timelines
- Keep existing subject timeline but add:
  - per-subject gap-duration summary strip (total non-stim time),
  - small multiples sorted by total delay.
- Goal: quickly detect sessions where waiting/interruption structure differs strongly.

### B) Emotion-Pair Radar For Bodyparts
- For each emotion pair, show a 9-bodypart radial profile using KS-D medians.
- One page with 6 pairs (fear-excluded mode) could show torso-vs-limb tradeoffs immediately.

### C) Baseline-Normalized vs Absolute Joint View
- Two aligned panels per metric:
  - absolute values,
  - fold-baseline.
- Same colors and same emotion ordering; only x-axis units change.
- Goal: avoid mentally comparing separate figures.

### D) Subject-Consistency Overlay
- For each bodypart + emotion pair:
  - plot median KS with an uncertainty band (for example IQR across subjects).
- Goal: separate strong effect size from unstable subject-level effects.

## 4) SAL/MAD Next-Step Plan (Immobility Windows)

Current status:
- speed has immobility-window arrays (`speedArrayImmobile`) wired into KS/CDF paths.
- SAL and MAD are available mainly as whole-window summaries in current plotting paths.

Proposed incremental path (non-algorithm first):
1. expose immobility-window SAL/MAD fields in output structs/tables (readability-first naming),
2. add plotting accessors to CDF/group plots analogous to speed options,
3. add run metadata flags showing whether SAL/MAD are whole-window or immobility-window derived,
4. only after review: propose inferential statistics for those fields.
