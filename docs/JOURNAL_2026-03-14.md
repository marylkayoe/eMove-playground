# Journal Notes (2026-03-14)

## 1) What Settled Today

- The project now has two genuinely useful presentation/exploration tools:
  - a subject-level micromovement browser for finding illustrative examples,
  - a group-level CDF browser for quickly testing emotion/bodypart/mode combinations.
- The browsing workflow is now much closer to how an actual analyst wants to work:
  - settings persist,
  - subject switching no longer destroys context,
  - exploratory iteration is fast enough to support figure design.
- The current evidence path should not force the claim that micromovement is
  always "better" at differentiating emotions than full movement.
  A more defensible framing is:
  - overt movement is more confounded by voluntary, strategic, or task-shaped behavior,
  - micromovement offers a complementary and potentially less voluntary window.

## 2) Figure-Making Lessons

1. Example-first figures are doing important work here.
   - The subject-level browser is useful not just for QC but for argument design.
   - For collaborators/funders, one strong example trace can be more persuasive
     than a dense grid of summary statistics.

2. Group summaries still matter, but they should support the story rather than
   carry the entire conceptual claim.
   - The new CDF browser is therefore best thought of as a comparison sandbox.
   - It helps answer:
     - which bodypart is most readable,
     - which metric scale is easiest to explain,
     - which emotion contrast is robust enough to show.

3. It is worth separating two distinct questions:
   - "Does a contrast appear in the data?"
   - "Why is this contrast scientifically interesting?"
   The second question may rely more on the logic of voluntary vs involuntary
   movement than on stronger discrimination alone.

## 15) 2026-03-29: Poster Reality Check

The poster is beginning to have a real structure now, but the figure logic is
still more fragile than it looks at first glance.

What feels solid:

- The project has a convincing central idea:
  there is an affect-relevant signal in the low-animation regime that is not
  simply "less motion."
- Single-subject example figures remain essential.
  They are doing conceptual work that no aggregated heatmap can do by itself.
- FEAR really does look like a special case.
  Treating it separately is not just a design preference; it prevents the
  non-fear summaries from degenerating into "distance from FEAR."

What felt slippery today:

- It is easy to make a figure look more decisive than the analysis deserves.
- Signed body maps, mixed pair-order conventions, and over-clever color coding
  all pushed the visuals toward apparent clarity while actually making them
  less trustworthy.
- I had to correct several instances where the visual framing changed more than
  the underlying data.

That is a useful warning.
This project is at a stage where the main danger is no longer "finding no
pattern," but "overstating a pattern by letting the figure language outrun the
 analysis."

### What I think is genuinely worth keeping

1. FEAR summary body maps
   - strong enough to be shown as a full-vs-LAR comparison
   - useful for telling the audience that some emotions are globally obvious
     in the body

2. Non-fear target body maps
   - `DISGUST`, `JOY`, `SAD` against non-fear comparators only
   - this is a better way to ask the subtler question

3. Subject example + pooled summary pairing
   - one example figure should introduce the intuition
   - one aggregated figure should say "this is not just that one person"

### Caveats that matter more than I first thought

- Browser micromovement views are still tied to precomputed immobile arrays.
  That is acceptable for now, but it must not be forgotten when figures are
  captioned.
- Residual-quality reassurance is still missing because the current Vicon CSV
  exports do not carry triangulation residual columns.
  That means we still cannot answer, from these exports alone, how large the
  micromovement signal is relative to residual reconstruction error.
- Body-map color scales are now being chosen for interpretability.
  That is fine, but it means the body maps are descriptive poster tools, not
  universal quantitative atlases.

### What I would do next, if this were a paper and not a poster deadline

- add a residual / mocap-quality analysis if a richer Vicon export can be obtained
- create one unified legend/caption language across:
  - single-subject example densities,
  - subject/pooled KS heatmaps,
  - bodypart summary maps
- decide explicitly which figures are:
  - inferential
  - descriptive
  - explanatory

That last distinction is increasingly important.
Right now the codebase can generate all three kinds of figures, but the poster
still risks blending them together.

### Practical poster advice to my future self

- Do not let the body maps carry more epistemic weight than they should.
- Use them to answer:
  - "where in the body is this emotion most resolvable?"
- Do not use them to imply:
  - mechanistic localization,
  - effect universality,
  - or direct comparability across every figure family.

The poster is now good enough that the next gains will come less from more
analysis and more from figure discipline.

## 8) All-Bodypart Regime-Distinctness Pass (Pooled Raw)

A broader pooled analysis across all bodyparts and all coded emotions was run in:

- `/Users/yoe/Documents/REPOS/eMove-playground/outputs/figures/regime_distinctness_20260314_195024`

The analysis asked:

- Are emotion relationships preserved between full-speed and micromovement regimes?
- Or do emotion contrasts reorder or flip sign, suggesting a distinct low-animation regime?

### Diagnostic logic

For each bodypart, using pooled raw samples across subjects:

1. compute emotion-specific pooled medians in:
   - full-speed regime
   - micromovement regime
2. compare the ordering of emotions between regimes using Spearman rank correlation
3. compute all 10 pairwise emotion contrasts and count how often the sign flips
   between regimes

If micromovement were only a downscaled shadow of full movement, one would expect:

- high rank agreement across emotions
- few or no pairwise sign flips

### Key result

That is **not** what the pooled upper-body results look like.

Strong regime change:

- `UTORSO`: `rho=0.10`, sign-flip fraction `0.50`
- `UPPER_LIMB_L`: `rho=0.10`, sign-flip fraction `0.50`
- `UPPER_LIMB_R`: `rho=0.10`, sign-flip fraction `0.50`
- `WRIST_L`: `rho=0.30` absolute / `0.10` normalized, sign-flip fraction `0.40-0.50`
- `WRIST_R`: `rho=0.10`, sign-flip fraction `0.50`
- `LTORSO`: `rho=0.10`, sign-flip fraction `0.50`
- `HEAD`: `rho=0.30` absolute / `0.20` normalized, sign-flip fraction `0.40`

More stable:

- `LOWER_LIMB_L`: `rho=0.90`, sign-flip fraction `0.10`
- `LOWER_LIMB_R`: `rho=1.00` absolute / `0.90` normalized, sign-flip fraction `0.00-0.10`

### Interpretation

This pooled pattern argues against a simple "same ordering, smaller amplitude"
account for the low-animation regime.

Instead, the results suggest:

- upper-body and head micromovements are not merely reduced versions of overt movement
- the affective geometry changes substantially between regimes in the upper body
- lower-limb motion appears more regime-stable, which is itself informative

That is a much stronger and more nuanced statement than "micromovement is better."

It suggests that:

- the low-animation regime may behave like a qualitatively different affective signal
  in upper-body/head channels
- while lower-limb motion may remain closer to the same underlying ordering across regimes

## 9) Subject-Level Follow-Up: More Stable Than The Pooled View

A subject-level follow-up was run in:

- `/Users/yoe/Documents/REPOS/eMove-playground/outputs/figures/regime_subject_level_20260314_195909`

This analysis used per-subject emotion medians for each bodypart and compared
full-speed vs micromovement within subjects.

### Key question

Does the pooled regime distinctness reflect a broadly consistent subject-level
effect, or is it amplified by aggregation?

### Main subject-level result

The subject-level picture is **more conservative** than the pooled one.

Across bodyparts:

- median subject-level Spearman rho is mostly `0.7-0.9`
- fraction of subjects with negative rho is low:
  - typically `0.00-0.07`
- median pairwise sign-flip fraction across subjects is modest:
  - roughly `0.11-0.26`

Most regime-sensitive bodyparts at subject level:

- `WRIST_L`: median subject rho `0.70`, median pair-flip fraction `0.26`
- `UPPER_LIMB_L`: `0.70`, `0.22`
- `UPPER_LIMB_R`: `0.70`, `0.22`
- `HEAD`: `0.70`, `0.20`

Most stable:

- `LOWER_LIMB_L`: `0.90`, `0.11`
- `LOWER_LIMB_R`: `0.90`, `0.15`

### Interpretation

This changes the claim in an important way.

What the data **does support**:

- the pooled upper-body micromovement regime is not trivially identical to the
  pooled full-motion regime
- some emotion-pair relations do flip, especially in upper-body channels
- wrists and upper limbs remain the most regime-sensitive bodyparts

What the data **does not yet support strongly**:

- a claim that most individual subjects undergo a wholesale regime inversion
- a claim that micromovement is universally "different" in the same dramatic
  way for every subject

The more accurate postdoctoral interpretation is therefore:

- the pooled affective geometry changes substantially between regimes,
  especially in upper-body channels
- but within-subject regime structure remains partly preserved
- therefore the low-animation regime is best framed as a **partially distinct,
  not wholly separate, affective regime**

### Why this is still interesting

That nuance is actually valuable.

It suggests:

- micromovement is not just noise,
- it is not merely a scaled-down copy either,
- but it also does not erase the underlying subject-level affective structure.

That makes the most defensible claim something like:

- "Micromovement reveals a complementary affective regime whose geometry is
  shifted relative to overt movement, especially in the upper body."

## 10) Late-Night Correction To My Own Thinking

I think I was still carrying too much of a "find the one killer plot" mindset.
That was a mistake.

The data is saying something interesting, but not in a cartoonishly simple way.
The temptation is to search for a figure that proves everything at once:

- micromovement is different,
- the upper body matters,
- disgust and joy reverse,
- pooled effects are strong,
- subject-level effects are still present,
- and the whole thing should be legible to an outsider in ten seconds.

That is too much load for one figure.

The better lesson from today is that there are actually *three* stories here,
and they should probably not be collapsed into one panel:

1. a **mechanistic / conceptual story**
   - overt motion can be strategic, stylized, and task-shaped
   - low-animation micromovement is interesting precisely because it is less
     obviously under voluntary expressive control
2. a **pooled-structure story**
   - upper-body emotion relationships reorganize between full and micro regimes
3. a **subject-level caution story**
   - this reorganization is real, but less dramatic within a typical subject

If I try to make one plot do all three jobs, the figure becomes muddy and I
start compensating with too much explanation.

## 15) Two Papers Worth Keeping In Mind

I wanted to resist the temptation to float too far away from the data, but it
also felt useful to check whether the broader literature gives this line of
thinking any backbone.

### Paper A: Hillman, Rosengren, Smith (2004)

Source:
- [Emotion and motivated behavior: postural adjustments to affective picture viewing](https://pubmed.ncbi.nlm.nih.gov/15019170/)

What I take from it:
- affective stimuli can modulate posture-related behavior, not just ratings or
  facial/autonomic measures
- the authors explicitly measured postural adjustment while people viewed
  emotional material
- the abstract reports sex-linked differences for unpleasant material, with
  directional postural shifts

Why it matters here:
- this supports the broad premise that small posture/body adjustments can be an
  affect-relevant channel, not just motor noise
- it does **not** validate our exact micromovement claim, but it gives the
  question legitimacy

My note to self:
- our work is richer than a classic body-sway paradigm because it uses
  full-body mocap and can ask where in the body regime shifts happen
- but it would be smart to reference this older postural-adjustment literature
  when motivating the project

### Paper B: Gea et al. (2014)

Source:
- [Viewing pain and happy faces elicited similar changes in postural body sway](https://pubmed.ncbi.nlm.nih.gov/25093727/)

What I take from it:
- different affective stimuli do not map onto a simple one-dimensional
  pleasant/unpleasant ordering in posture
- the abstract reports that both happy and pain faces increased anterior-
  posterior sway relative to neutral

Why it matters here:
- this is actually very congenial to what we are seeing
- it suggests that emotion effects in body fluctuation are not necessarily
  monotonic or intuitively ordered
- in other words, a surprising regime-dependent reordering in our data would
  not be absurd

My note to self:
- this paper is a useful antidote to any naive expectation that there should be
  one obvious ranking of emotions by "more" or "less" movement

### Paper C: Chen et al. (2023)

Source:
- [SMG: A Micro-gesture Dataset Towards Spontaneous Body Gestures for Emotional Stress State Analysis](https://link.springer.com/article/10.1007/s11263-023-01761-6)

What I take from it:
- there is already a modern line of work explicitly arguing that subtle and
  spontaneous body movements can carry hidden emotional or stress-related
  information
- this paper frames micro-gestures as subtle, spontaneous movements with
  psychological meaning and argues they can be more reliable than ordinary
  facial expression for hidden emotional information

Why it matters here:
- it does not solve our scientific question, but it tells me this is not a
  bizarre conceptual direction
- it also highlights a useful distinction:
  there is a difference between **acted expressive emotion** and
  **subtle spontaneous body movement under internal state**

My note to self:
- our project may be most interesting precisely where it is *not* about
  theatrical body language, but about low-amplitude structure in immersive
  emotional context

## 17) New Concern: Are The Many Reversals Just Noise?

Today’s combined pooled pair-scatter did something useful: it made the pattern
look visually coherent.

That is good for seeing structure, but it also creates a danger:

- a clean pooled picture can seduce me into treating a fragile pattern as a
  robust one

The concern is now very concrete:

- perhaps many bodyparts appear to reverse on many emotion pairs simply
  because pooled aggregation sharpens weak and noisy pairwise differences
- perhaps near-zero contrasts are being pushed across quadrants by noise
- perhaps the pooled-raw summary is overemphasizing subjects with more usable
  samples

So the right scientific question is no longer:

- "Can I make the reversal picture look good?"

It is:

- "How stable is the reversal picture once I respect subjects as subjects?"

## 18) Plan For The Stability Check

I want to answer that concern with three specific analyses:

1. **Subject-consistency heatmap**
   - for each bodypart x emotion pair:
   - what fraction of subjects actually show a sign reversal?

2. **Bootstrap-over-subjects stability**
   - resample subjects with replacement
   - aggregate subject-level pair contrasts
   - ask how often the aggregate still lands in a reversal quadrant

3. **Aggregation comparison**
   - compare:
     - pooled-raw pairwise contrast
     - subject-median aggregate pairwise contrast
   - if those disagree often, the pooled result is fragile

What would reassure me:

- some bodypart-pair combinations show high subject flip fraction
- bootstrap stability is clearly above chance for the same combinations
- pooled and subject-median aggregate pictures mostly agree in sign/quadrant

What would worry me:

- pooled reversals look dramatic but subject-level flip fractions stay around
  coin-flip values
- bootstrap stability is weak
- pooled and subject-median aggregation disagree often

If the worrying version happens, the correct response is not to hide it.
The correct response is to narrow the claim.

## 19) Practical Figure Consequence

If the reversal picture turns out to be only moderately stable, then tomorrow's

- "many bodyparts reverse on many pairs"

Instead it should say something more precise, for example:

- "upper-body emotion geometry changes across regimes, but only some
  pair/bodypart combinations are stably reversal-like"

That would still be interesting. It would also be a more serious result.

## 20) Reversal Filters: What Changed, What Did Not

I finally tested the obvious discomfort directly: perhaps a lot of the
subject-level reversals are just weak near-zero contrasts hopping across zero.

I compared three ways of counting subject-cells:

1. **Reference**
   - count all comparable subject cells

2. **Dead-zone**
   - only count a subject-cell if at least one regime has a contrast large
     enough to escape a practical near-zero zone

3. **Bootstrap-CI**
   - only count a subject-cell if the full or micro contrast bootstrap CI
     excludes zero

So the broad picture is not being created from completely trivial near-zero
cases. That is reassuring.

But the filters are still useful, because they tell me where the fragile cells
are. The biggest dead-zone changes tend to hit cells like `JOY-SAD` or some
`NEUTRAL-*` comparisons, not the disgust-centered torso cells that already
looked most convincing.

That sharpens the story further:

- the disgust-centered torso effect is not disappearing under reversal-specific
  inclusion rules
- some of the more generic-looking reversals really are weaker and more
  sensitive to how I define an interpretable subject-level contrast

## 21) What I Think This Means For Tomorrow

I do **not** want tomorrow's figures to imply:

- "all sorts of emotions reverse all over the body"

The better message now is:

- "regime dependence exists"
- "it is not just a lower-limb or whole-body threshold artefact"
- "the clearest stable part of the picture is disgust-centered and
  torso-weighted"

There is an awkward but important tension now:

- the visually most exciting plot is the pooled combined scatter
- the scientifically most trustworthy plot is the stable-cell summary

So the presentation has to respect both.

My current instinct is:

1. start with a human-readable single-subject example
2. show the pooled combined scatter as the intuition-generating picture
3. immediately follow with the stable-cell heatmap so nobody can accuse us of
   hiding the fragility question

That sequence feels honest and still impactful.

## 22) Today Turned Out To Be About Trust

The biggest lesson today was not a numerical one. It was semantic.

I kept thinking I was testing the data, but a surprising amount of the friction
came from testing the figures.

Two things repeatedly changed the apparent story without changing the underlying
numbers:

1. **pair order**
   - `DISGUST-JOY` and `JOY-DISGUST` are not harmless relabelings when the
     plot encodes a signed contrast.
   - The visual quadrant can flip while the biology stays the same.

2. **aggregation level**
   - pooled raw contrast is not the same as subject-level contrast
   - and neither is the same as the bootstrap stability of a pooled effect

That seems obvious when written out calmly, but in live figure iteration it was
easy to drift into treating them as interchangeable.

So the real progress today was not "we found the definitive plot."
It was that the unsafe shortcuts became visible.

## 23) Where I Actually Land Tonight

I do think there is a real signal here.

But I no longer believe the broad version:

- not "many bodyparts reverse for many emotion pairs"
- not "micromovement just cleanly reorganizes emotion space everywhere"

The version I trust now is narrower and, in a way, more interesting:

- the clearest stable regime-dependent change is **disgust-centered**
- it lives mostly in **torso** signals
- `HEAD` helps, but it is not carrying the whole story alone
- legs mostly sit there like a useful reminder that not every bodypart should
  be expected to cooperate

That is a better result than a cartoonishly dramatic one. It sounds more like a
real dataset and less like an overfit slide.

## 24) Why `SC3001` Still Wins As The Subject Example

I did spend real time trying to find a more hygienic example than `SC3001`.

And that instinct was good. `SC3001` is not perfect:

- the immobility is not beautifully sustained
- some of the low-speed bouts are shorter and messier than I would ideally want

But several alternatives had the opposite problem:

- more sustained immobility
- weaker or less elegant reversal story

And at some point the figure has to serve the concept.

`SC3001` still does that best:

- the head/upper-torso/lower-torso relationship is readable
- the disgust-vs-other ordering change is visible
- and the single-subject example actually looks like a mechanism illustration
  rather than a random anecdote

So I am accepting the tradeoff consciously rather than pretending it is ideal.

## 31) Baseline-Disruption Work Was Stopped By The Filesystem, Not The Data

Today’s first real obstruction on the long-baseline disruption question was not
conceptual. It was the machine.

The repo had silently accumulated a huge mass of stale git pack temp files in:

- `.git/objects/pack/tmp_pack_*`

The amount was not trivial. It was catastrophic enough to make ordinary work
look scientifically broken:

- MATLAB could not write CSV outputs
- shell here-docs stopped working
- even small file edits started to fail

That is a good reminder that “analysis failure” and “environment failure” are
not the same thing, and I need to keep that distinction sharp when something
starts behaving strangely.

The fix was straightforward once identified:

- inspect free space
- inspect repo-local storage pressure
- remove stale `tmp_pack_*` files

The real lesson is procedural:

- future workspace cleanup should explicitly include stale git temp packs, not
  just old figure exports and scratch QC folders

If I do not write that down, I will end up relearning it the hard way later.

## 25) What I No Longer Want To Show

There were several figure ideas today that were not wrong, but became less and
less trustworthy the more I looked at them.

I do **not** want to lead with:

- signed stick figures where color tries to encode both magnitude and
  direction
- pooled scatters whose quadrant logic depends on mixed pair-order conventions
- any plot that can only be defended with a five-minute explanation of why the
  signs came out the way they did

That is not because the data are bad.
It is because the visual semantics are too fragile.

If a viewer can misread the figure faster than I can correct them, the figure
is not ready.

## 26) What Feels Worth Polishing Tomorrow

## 27) Caveat I Want To Keep Honest

We are still not addressing inter-individual variability in any deep way.

That is not laziness. It is partly a dataset-size issue and partly a time issue.

So for now the honest scope is:

- a selective group-level story
- with one illustrative subject-level mechanism example
- not a claim that every individual expresses the same reversal geometry

That caveat should stay attached to the work even if the final slides become
much cleaner than the process felt.

## 28) Browser Boundary Became Clearer

Today's browser work was useful precisely because it forced a boundary.

The subject-density browser is now genuinely helpful for:

- testing whether a contrast is visually readable,
- deciding whether density or CDF is the better presentation choice,
- checking whether a long tail is a real feature or just ruining the axis,
- getting a fast pairwise KS overview for one subject or a pooled exploratory view.

But this same convenience creates a risk: once a browser becomes flexible
enough, it starts to feel like an analysis pipeline in its own right.

I do not want that.

So the line I want to keep is:

- browsers = exploration, intuition, figure design, QC
- main scripts = reporting and actual result summaries

The pooled browser views are especially tempting because they look immediate and
clean, but they are still pooled-sample exploratory views, not the same thing
as a subject-aggregated population statistic. That distinction needs to remain
explicit in both the docs and my own thinking.

## 29) What The New Browser Work Suggests For Next Steps

A few practical ideas feel more settled now:

1. The density/CDF browser is enough to scout presentation figures without
   writing a custom script every time.

2. The heatmap view is useful only after simplifying it.
   - one semantic per cell,
   - one shared color scale per figure,
   - stars only for significance.

3. The browser should stay honest about micromovement.
   - right now it uses the precomputed immobile arrays,
   - which is acceptable as long as we say so clearly,
   - but if threshold flexibility becomes important later, it needs to be
     implemented properly from the raw speed arrays.

4. The next figure work should probably be smaller, not larger.
   - a few very clean panels,
   - rather than new sprawling summary figures.

## 30) Personal Note

There is a pattern in this project that keeps repeating:

- the first figure is usually too ambitious,
- the second figure is more technically informed but visually worse,
- the third figure becomes useful only after removing half of the cleverness.

That is frustrating in the moment, but it is also probably the right process.

The nice thing is that the codebase is now reaching a point where those cycles
are faster. I can check an idea in the browser, reject it, and move on, instead
of needing a new script for every test. That is real progress even if the final
scientific story is still being tightened.
