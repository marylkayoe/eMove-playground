# Journal Notes (2026-05-03)

## Repository Safety Cleanup

Today was not about new analysis. It was about making the repository safe to
work in again.

The main problem was that Codex Desktop became dangerous once large generated
and local-only files were still living inside the working tree. That produced
runaway Git pack growth and a repository state that was technically recoverable
but operationally unhealthy.

The useful lesson is straightforward:

- the repo should hold curated source, docs, and small tracked resources
- large derived outputs should live in local data storage
- exploratory agent artifacts should live under `scratch/`

## What Changed

- `.git` was brought back to a sane size after cleanup
- `outputs/` was moved out of the repo to local data storage
- `legacy_resultCellSingles.mat` was moved out of the repo to local data
  storage
- `AGENTS.md` now defines the repository safety rules and maintenance workflow
- `scratch/` is now the explicit required home for exploratory agent-generated
  files

## Why This Matters

This is not only about disk usage.

It is about keeping the project legible to both humans and tools.

Once temporary figures, MAT artifacts, caches, and local environments start
living too close to tracked source, the repo stops being a clean statement of
the project and starts becoming an accidental warehouse. That makes Git
operations less predictable and makes agent tooling more brittle.

## Small Validation After Cleanup

Codex Desktop passed a small scratch-workflow test after the cleanup.

That matters because it shows the current rules are not merely prohibitions.
They also define a workable path:

- exploratory work can still happen
- it just has to happen under `scratch/`
- and it has to be easy to inspect, keep, promote, or delete intentionally

## Practical Rule To Keep

If the output is exploratory, intermediate, generated, or local-only, it should
start in `scratch/` or local data storage, not in tracked project folders.
