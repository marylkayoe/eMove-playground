# Journal Notes (2026-03-14)

## 1) What Settled Today

- The project now has two genuinely useful presentation/exploration tools:
  - a subject-level micromovement browser for finding illustrative examples,
  - a group-level CDF browser for quickly testing emotion/bodypart/mode combinations.
- The browsing workflow is now much closer to how an actual analyst wants to work:
  - settings persist,
  - subject switching no longer destroys context,
  - exploratory iteration is fast enough to support figure design.
- The current evidence path should not force the claim that micromovement is
  always "better" at differentiating emotions than full movement.
  A more defensible framing is:
  - overt movement is more confounded by voluntary, strategic, or task-shaped behavior,
  - micromovement offers a complementary and potentially less voluntary window.

## 2) Figure-Making Lessons

1. Example-first figures are doing important work here.
   - The subject-level browser is useful not just for QC but for argument design.
   - For collaborators/funders, one strong example trace can be more persuasive
     than a dense grid of summary statistics.

2. Group summaries still matter, but they should support the story rather than
   carry the entire conceptual claim.
   - The new CDF browser is therefore best thought of as a comparison sandbox.
   - It helps answer:
     - which bodypart is most readable,
     - which metric scale is easiest to explain,
     - which emotion contrast is robust enough to show.

3. It is worth separating two distinct questions:
   - "Does a contrast appear in the data?"
   - "Why is this contrast scientifically interesting?"
   The second question may rely more on the logic of voluntary vs involuntary
   movement than on stronger discrimination alone.

## 3) Practical Next-Step Visualization Ideas

### A) Same-Subject Matched Contrast Card
- One subject.
- Same bodypart(s).
- `NEUTRAL` and `DISGUST` shown in aligned windows.
- Top row: position traces.
- Bottom row: speed traces with immobility shading.
- Goal:
  - make the low-speed structure visible without claiming that the effect is
    universally stronger in all metrics.

### B) Overt-vs-Micro Split Panel
- Left:
  - full-speed group summary.
- Right:
  - micromovement-only group summary.
- Caption language should emphasize:
  - different analysis window,
  - different interpretive value,
  - not necessarily a "winner".

### C) Bodypart Triage Pass
- Use the CDF browser to decide which 2-3 bodyparts are worth showing at all.
- Likely candidates remain:
  - `UTORSO`
  - `WRIST_L`
  - `WRIST_R`
  - possibly `HEAD`
- The objective should be readability plus conceptual fit, not only the largest p-value.

## 4) Methods / Tooling Thoughts

- The CDF browser now supports:
  - multi-emotion overlays,
  - multiple bodyparts,
  - median vs raw modes.
- This is enough to do presentation scouting without writing new scripts for every idea.
- If this tool continues to prove useful, the next practical additions would be:
  1. export button from the UI,
  2. saved presets,
  3. compact plot styling mode for slide-ready output.

## 5) SAL / MAD Notes

- SAL and MAD remain scientifically interesting, but they are not yet as
  reachable in the current exploratory UI as speed.
- Before extending claims around micromovement with these metrics, the project
  likely needs:
  1. a clear definition of SAL/MAD computed on whole stimulus windows vs
     immobility-only windows,
  2. plotting access in the same browser style as speed,
  3. one or two reference figures that show what a meaningful SAL/MAD effect
     actually looks like.

## 6) Open Question To Carry Forward

- Which figure is the true center of gravity for collaboration pitches:
  - a strong single-subject matched example,
  - a bodypart-specific group summary,
  - or a combined example + summary panel?

The tooling built today should let that decision be made from the data rather
than by speculation.
